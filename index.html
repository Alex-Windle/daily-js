Closure

"When a function remembers and accesses its lexical scope even when that function is executing outside its lexical scope." -YDKJS

"If you execute a function inside a function, a closure is created. Closure could return a function. Closures saves the state of the outer function variables (private vars) but do not expose them. You can hide implementation details with "encapsulation"." -ThatJSDude

"Closures are frequently used in JavaScript for object data privacy, in event handlers and callback functions, and in partial applications, currying, and other functional programming patterns." -Eric Elliott

[ {firstName: 'Joe', lastName: 'Blow', age: 42, role: 'clerk'}, {firstName: 'Mary', lastName: 'Jenkins', age: 36, role: 'manager'} ]

Note that the input may have a different number of rows or different keys than the given sample.
